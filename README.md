# AI-Powered Income Protection for Gig Workers

# Requirement & Persona-Based Scenario

Our solution targets gig workers such as Zomato, Swiggy, and delivery partners who face income
loss due to external disruptions like weather, pollution, or curfews.

1 Persona: Food delivery partner working daily for earnings
2 Problem: Sudden disruptions reduce daily income
3 Solution: AI-based system predicts, prevents, and protects income

# Workflow of Application

1 User onboarding and profile setup
2 AI risk assessment and weekly premium calculation
3 Real-time monitoring of disruptions (weather, AQI, etc.)
4 Automatic trigger detection
5 Auto claim generation and fraud validation
6 Instant payout via UPI

# Weekly Premium Model & Parametric Triggers

The platform follows a weekly micro-insurance model aligned with gig worker earnings.
1 Premium range: ■10–■50/week based on risk
2 Dynamic pricing using ML models
3 Triggers:
4 Heavy rain (>50mm) → payout
5 AQI > 300 → payout
6 Curfew / strikes → payout
7 App downtime → payout

# AI/ML Integration

1 Risk Prediction Model using historical weather and activity data
2 Dynamic Pricing Model using regression
3 Income Loss Estimation using past earning patterns
4 Fraud Detection using anomaly detection models
5 AI Agent for personalized guidance and alerts

# Unique Differentiation

1 Predict → Prevent → Protect approach
2 AI suggests best working hours to avoid loss
3 Dynamic payout based on actual income loss
4 Community-based disruption detection
5 Voice and WhatsApp integration for accessibility

# Tech Stack

1 Frontend: React / Next.js
2 Backend: Node.js / Flask
3 Database: MongoDB
4 AI/ML: Python (Scikit-learn, TensorFlow)
5 APIs: Weather, AQI, Maps
6 Payments: Razorpay / UPI sandbox

# Data Security & Privacy

1 End-to-end encryption (TLS + AES-256)
2 Zone-based location tracking (no exact GPS)
3 Role-based access control
4 Fraud detection for security
5 Compliance with DPDP principles
